直接写入字符串

如果想要写入字符串，则需要使用到Writer抽象类，该类有两个实现类，分别是OutputStreamWriter和FileWriter，其中前者在实例化对象的时候需要传入OutputStream类的实例对象（OutputStream对象包含了文件的路径，所以在不能直接拿到文件路径，但是有OutputStream对象的情况下，就可以使用OutputStreamWriter类），而后者则可以直接通过文件路径来实例化。

使用缓冲

直接写字符串使用Writer，但是读字符串就不会直接使用Reader，尽管也会使用到类似于OutputStreamWriter的InputStreamReader或类似于FileWriter的FileReader，但是一般的读字符串操作会在此基础上为了提升效率而再包裹一层 Buffer ，即BufferedReader类。

使用缓冲读字符串

需要注意，Reader类中并没有readLine()，所以如果需要读文本，尤其是需要依靠换行符来将每一部分文本分开的时候，就可以使用BufferedReader。BufferedReader类的构造函数接收一个Writer对象，所以在这里既可以传递InputStreamReader的实现，也可以传递FileReader的实现。随后，BufferedReader对象通过调用readLine()读取每一行文本（遇到\n、\r、\r\n中的任何一个再或者是到了 EOF ）。另外，在BufferedReader类中有一个静态成员变量defaultCharBufferSize值为8192，其实也就是指明了BufferedReader对象的默认缓冲大小。

NIO

与传统 I/O 相比，NIO 具有几个特点：

  • 没有 Stream ，取而代之的是「Channel」。Stream 是单向的，单向读或写，而 Channel 是双向的。
  • 强制使用 Buffer ，Buffer 本身可以被操作。
  • 网络交互支持非阻塞，文件交互仍然阻塞。

读

首先要获取 Channel ，然后创建 Buffer ，把从 Channel 读出来的数据写入 Buffer 中。

到了这里会下意识地想去调用 Buffer 的 read 相关方法，但遗憾的是ByteBuffer并没有read()或者是readLine()。

首先在ByteBuffer中有 3 个重要的参数：

  • capacity → 决定了 Buffer 的大小。
  • position → 下标，或者说是指针，指向用于读或写的字节，初始位置为0。
  • limit → 指针的上限，即指针在移动的时候不可超过这个位置，默认与capacity相等。

在一条水平的轴上，capacity 始终处于最右端，而 limit 因为默认等于 capacity ，所以它在一开始的时候会和 capacity 处于最右端。Buffer 的读写都是按照从左往右的顺序，在channel.read(buffer);的时候，实际上是在往 Buffer 中写数据，所以 position 会从左往右移动，例如写了两个字节分别是a和b，那么此时 position 已经发生变化，接下来就需要从 Buffer 中读取已经写入的数据，但是在这之前，需要先：
