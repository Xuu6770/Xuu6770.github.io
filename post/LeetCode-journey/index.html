LCP 66. 最小展台数量

题目链接：https://leetcode.cn/problems/600YaG/
题目大意：给定一个由字母组成的字符串数组，输出每个字母在单个字符串中出现最多的次数之和。

输入：demand = ["acd","bed","accd"]
输出：6

解释：

  • 字母a在第一个字符串和第三个字符串中都只出现了一次，记为 1 。
  • 字母b在第二个字符串中出现了一次，记为 1 。
  • 字母c在第一个字符串中出现了一次，但是在第三个字符串中出现了两次，所以记为 2 。
  • 字母d在三个字符串中都有出现，但是每次出现最多只有一个，所以记为 1 。
  • 字母e只在第二个字符串中出现了一次，记为 1 。

故合计为 6 。

思路及方案：

  1. 创建一个 Map 用于存储每个字母以及其出现的次数。
  2. 遍历给定的字符串数组，将数组中的每个字符串单独拿出来，使用 Kotlin 的标准库函数groupingBy()和eachCount()来返回另一个 Map 对象dayDemand，这个 Map 对象记录着在当前字符串中出现的每个字母以及出现的次数。
  3. 遍历dayDemand这个 Map 对象，将其中的数据与map变量中的数据进行比较，保留出现次数最多的结果。
  4. 将map变量中的所有值加起来并返回。

思考与总结：

  • mutableMapOf()是 Kotlin 标准库中的函数，用于创建可变的 Map 对象。
  • groupingBy()是 Kotlin 标准库中的函数，和groupBy()一样，都可以用于对集合进行分组操作。
    • groupBy()会将集合中的元素根据给定的 keySelector 函数进行分组，并返回一个 Map 对象，其中 key 为分组依据，value 为分组后的元素列表。
    • groupingBy()则返回一个Grouping对象，该对象可以进行更高级别的分组操作，例如使用eachCount()、fold()、reduce()等函数进行聚合操作，最终返回一个 Map 对象。Grouping对象是一个惰性序列（与普通序列不同，惰性序列只在实际需要时才计算元素），故而在进行大量数据分组时，使用groupingBy()可以提高性能。
  • Kotlin 对[]进行了运算符重载，所以可以直接用[]操作符来访问集合中的元素（Java 只能访问数组中的元素），例如map[type]。具体来说，就是在类中定义一个名为operator fun get()的函数，从而允许类的实例使用[]操作符来访问元素，而定义了operator fun set()，就可以通过键来设置对应的值。

1. 两数之和

题目链接：https://leetcode.cn/problems/two-sum/
题目大意：给定一个整型数组和一个整型变量，在数组中找到两个元素，这两个元素的和等于给定的这个整型变量。

输入：nums = [2,7,11,15], target = 9
输出：[0,1]

解释：nums[0] + nums[1] == 9 ，返回 [0, 1] 。

思路及方案：

作为题库的第一道题并不算太难，用万能的 Map 就能比较好地解决。

  1. 创建一个 Map 用于存储数组中的元素（key）以及对应的下标（value）。
  2. 遍历这个数组，对于每个元素，先算出目标整型与它的差值。
  3. 判断 Map 中是否存在这个差值（如果 Map 中存在一个 key 等于这个差值，也就对应说明了数组中是存在一个元素能与当前遍历的元素相加得到目标整型的），如果存在，就返回这个差值（key）对应的数组下标（value）。
  4. 否则，就将这个差值以及其对应的下标存入到数组中。
  5. 最后如果找不到结果，就抛出异常。不过这一步其实是多余的，因为题目说了，所有的测试用例都有答案，这里这么写是为了通过编译。

思考与总结：

  • (index, num) in nums.withIndex()这种写法的意义是：遍历这个数组的元素，同时拿到元素对应的下标，所以尽管数组中的元素都是「一个一个」遍历的，但是还是要额外有一个index参数用来接收数组下标。同时需要注意，参数index和num的位置不可调换。
  • 在 Kotlin 中，并没有for (int i = 0; i < nums.length; i++)这样的写法，不过类似的逻辑，可以这么写for (index in nums.indices)。其中indices是一个只读属性，它返回一个表示数组索引范围的整数区间，这个区间包含了从 0 到数组长度减 1 的所有整数值。
  • 在 Kotlin 中，可以通过contentToString()来打印数组，这样就不需要手动去遍历来输出结果。以上面的代码为例，可以这么写：println(twoSum(intArrayOf(2, 7, 11, 15), 9).contentToString())，打印结果为[0, 1]。

136. 只出现一次的数字

题目连接：https://leetcode.cn/problems/single-number/
题目大意：在一个非空整数数组nums中，除了某个元素只出现一次以外，其余每个元素均出现两次。找出并返回那个只出现了一次的元素。另外，算法必须实现线性时间复杂度且只使用常量额外空间。

输入：nums = [2,2,1]
输出：1

思路及方案：

  • 「除了某个元素只出现一次以外，其余每个元素均出现 两次 」，再加上题目后面额外的两个要求，整体给人的感觉就像是根据答案出题一样，完全就是为「异或运算」准备的。
  • 异或运算是一种二进制运算，用符号^表示。其规则为对于两个二进制位，若相同则结果为0，若不同则结果为1。例如：
    • 1^0=1
    • 0^1=1
    • 1^1=0
    • 0^0=0
    • 假设有两个十进制数 10 和 15 ，在进行异或运算前，需要先分别将其转换为二进制：1010 和 1111，再根据上面的规则，可以得到它们的异或值为 0101 ，转换回十进制即 5 。
  • 使用异或运算后会发现，但凡出现了两次了数字，最终都会相互抵消，那么剩下的那个数字自然就是结果了。

思考与总结：

  • 在代码中，使用了xor来进行异或运算，xor是通过函数重载来实现的操作符。在 Java 中，则是使用^符号来进行异或运算。
  • 在xor的签名中可以看到它被infix修饰。infix修饰符用于将函数声明为「中缀函数」，中缀函数可以被放置在两个操作数之间，例如 1 + 2 ，其中 + 是操作符，1 和 2 是操作数。要定义一个中缀函数，需要满足 3 个条件：
    • 该函数必须是成员函数或扩展函数。
    • 该函数必须只有一个参数。
    • 参数不能接受可变数量的参数，并且不能有默认值。
  • 类似的中缀函数还有and、or，它们都是Int类的成员函数。
  • 上面提到的「可变数量的参数」，实际上指的是一个被vararg关键字修饰的参数，在调用函数时，对于这样的参数，就可以不显式地使用数组或集合来传递多个同类型的参数。具体来看个例子：

387. 字符串中的第一个唯一字符

题目链接：https://leetcode.cn/problems/first-unique-character-in-a-string/
题目大意：给定一个字符串s，找到它的第一个不重复的字符并返回索引。如果不存在，则返回-1。

输入: s = “leetcode”
输出: 0

思路及方案：

比较简单，果断地用万能哈希表解决罢。

  1. 创建一个 Map 用于存储字符串中的每个字符（key）以及对应的出现次数（value）。
  2. 遍历这个字符串，在表中修改每个字符的出现次数。
  3. 再次遍历这个字符串，对于每一个字符，判断其在表中对应的 value ，如果为1则返回这个字符的下标。
  4. 如果遍历结束程序都没能返回，说明不存在只出现了一次的字符，故而返回-1。

思考与总结：

没什么需要额外记录的，代码中的很多用法都在前面的题目中有解释，感觉 Kotlin 的语法对开发者来说还是蛮友好的。
