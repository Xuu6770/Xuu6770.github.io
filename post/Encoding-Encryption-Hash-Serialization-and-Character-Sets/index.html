编码、加密、Hash、序列化和字符集

对称加密和非对称加密、Base64 、序列化、哈希和字符集。

密码学

古典密码学

起源于古代战争，在战争中，为了防⽌书信被截获后重要信息泄露，⼈们开始对书信进⾏加密。加密方式主要有：

  • 移位式加密。如密码棒，使⽤布条缠绕在⽊棒上的⽅式来对书信进⾏加密。尽管很古老，但是已经具备加密的两个元素，一是「加密算法」：将布条对木棒进行缠绕后再书写。二是「密钥」：⽊棒的尺⼨。
  • 替换式加密。使用字母表和码表，表示字母表中的字母应该如何被码表中的替换。加密算法：替换⽂字，密钥：替换的码表。

现代密码学

现代密码学利用数学原理，直接针对二进制数据进行加密与解密。因为文本文件本身也属于二进制文件，因此也可以进行加密和解密。现代密码学还分为「对称加密」和「非对称加密」。

对称加密

对称加密在古典密码学中对应的原型是替换式加密类似：即对⽂字进⾏规则化替换来加密，对密⽂进⾏逆向的规则化替换来解密。简单来说，对称加密就是通信双⽅使⽤同⼀个密钥，加密方使⽤加密算法和密钥加密，形成密文，解密方使⽤解密算法配合密钥来进⾏解密。加解密方法不一样，密钥一样。

现代密码学通过网络传输，在传输过程中可能经历了很多个节点，密文和密钥都增加了被截获的可能，故而现代密码学需要更注重反破解。

经典算法：

  • DES，56 位密钥，因密钥太短（容易被穷举得到答案）而被弃用。
  • AES，128 位、192 位、256 位密钥。

破解思路：先拿到⼀组或多组 原⽂-密⽂ 对。然后设法找到⼀个密钥，这个密钥可以将这些 原⽂-密⽂ 对中的原⽂加密为密⽂，以及将密⽂解密为原⽂，即为成功破解。

反破解：

  • 让破解者找不到⽐穷举法（暴⼒破解法）更有效的破解⼿段。
  • 穷举法所消耗的时间⾜够⻓（例如数千年）。
  • 密钥不能泄漏。

非对称加密

使⽤公钥对数据进⾏加密得到密⽂，使⽤私钥对数据进⾏解密得到原数据。简单来说，加密解密用同一套算法，不同的密钥，加密用公钥，解密用私钥。⾮对称加密使⽤的是复杂的数学技巧，在古典密码学中没有对应的原型。

使⽤⾮对称加密通信，可以在不可信⽹络上将双⽅的公钥传给对⽅，然后在发消息前分别对消息使⽤对⽅的公钥来加密和使⽤⾃⼰的私钥来签名，做到不可信⽹络上的可靠密钥传播及加密通信。例如 A 需要加密，那么只需要 B 创建一个「公钥 B」和「私钥 B」，然后将「公钥 B」发送给 A ，A 利用「公钥 B」对消息进行加密，然后再把加密后的消息发送给 B ，B 再用「私钥 B」解密即可。进一步说，「公钥 B」被称为「公钥」，表明了这个密钥就是公开的，任意一方想往 B 发送加密信息，只需要拿到这个密钥然后对消息进行加密就行了，最终也只有 B 能通过自己的「私钥」来解开密文。很明显，对称加密做不到，因为对称加密的加密和解密必须要求双方都持有同一个密钥，这个密钥就必须要在不可信网络上传输，从而被泄漏。

此外非对称加密还有一个数字签名的用法。前面提到非对称加密使用公钥加密，使用私钥解密。但实际上，私钥也可以用来加密，私钥加密后形成的密文，可以用其对应的公钥进行解密，又因为公钥和私钥总是成对生成的，并且在生成的时候就已经确定了谁是公（公钥往往更容易被计算出来，例如比特币的椭圆曲线加密算法，其公钥就是通过私钥计算得出）谁是私且不可再被修改，故而可以起到了一个身份验证的作用。例如 A 利用私钥签名（加密）了一段数据，此时 B 和 C 用公钥成功解密（验证）了数据，又因为私钥只有 A 持有，所以可以断定这个加密行为是 A 做的。

此外，尽管通过单方的公钥和私钥已经完成了防偷窥的效果，即通信内容不会被外人知道，但是却无法防止伪造的情况。详细点说就是，A 和 B 在互相发送公钥的过程中，公钥可能会被中间人 C 获取，从而 C 可以利用公钥伪造一条消息然后继续传输。所以 A 和 B 在互相发送公钥供对方加密消息的基础上，还需要用自身的私钥对消息进行签名，以保证对方在接收到消息的时候可以认证其接收到的消息是由己方发出。再简单点说，我给你发消息，我要用你的公钥加密消息，用我的私钥签名，发给你以后，你要用你的私钥解密，用我的公钥验证签名。

事实上，签名在实际的使用过程中会附带一个哈希的过程。前面提到签名实际上也是对消息的一次加密，只不过加密使用的是发送方的私钥，而加密的对象实际上是这个消息的哈希值。因为如果消息的体积很大，那就要在加密消息的基础上还要对消息进行签名，从而造成需要传输的数据量翻倍。但是哈希值很小，对消息进行哈希，然后再对哈希值进行签名，接收方在收到签名后使用发送方的公钥验证签名得到原哈希值，然后再用自己的私钥解密消息，然后再获取消息的哈希值，最后将两个哈希值一对比，就能保证这个数据是发送方发出的。如此一来，非对称加密的一次传输过程就算完成了。

经典算法：

  • RSA，可⽤于加密和签名。
  • DSA，仅⽤于签名，但速度更快。

非对称加密的优缺点：

  • 优点：可以在不安全⽹络上传输密钥。
  • 缺点：计算复杂，因此性能相⽐对称加密差很多。

破解思路：

  • 和对称加密不同之处在于，⾮对称加密可以很轻松地获得公钥从而很轻松地制造密文。所以，⾮对称加密的关键只在于，如何找到⼀个正确的私钥，可以解密所有经过公钥加密过的密⽂。找到这样的私钥即为成功破解。
  • 由于⾮对称加密的⾃身特性，怎样通过公钥来推断出私钥通常是⼀种思路（例如 RSA），但往往最佳⼿段依然是穷举法，只是和对称加密破解的区别在于，对称加密破解是不断尝试⾃⼰的新密钥是否可以将⾃⼰拿到的原⽂-密⽂对进⾏加密和解密，⽽⾮对称加密是不断尝试⾃⼰的新私钥是否和公钥互相可解。

反破解：

和对称加密⼀样，⾮对称加密算法优秀的标准同样在于，让破解者找不到⽐穷举法更有效的破解⼿段，并且穷举法的破解时间⾜够⻓。

密码学密钥和登录密码

密钥和密码是不一样的：

  • 名称不一样，密钥原文是key，密码原文是password。
  • 场景不一样，密钥⽤于加密和解密，如果密钥丢失，就代表整个数学运算的解题关键丢失了，无法算出结果。密码的英文直译实际上是「通行证」，例如⽤户在进⼊⽹站或游戏前的登录环节，账号和密码实际上是起到身份验证的作用，系统只需要比对密码和当初设置的是否一致，跟数学没关系。
  • 目的不一样，密钥的目的是保证数据不被他人读懂。密码是服务提供方对用户账户的一个保护，只有在认证成功时才提供服务。
  • 焦点不一样，密钥的焦点是数据，密码的焦点是身份。

Base64

二进制数据，⼴义来说所有计算机数据都是⼆进制数据，狭义来说⾮⽂本数据即⼆进制数据。而 Base64 是可以将⼆进制数据转换成由 64 个字符组成的字符串的编码算法。64 位，分别是 26 个英文字母的大小写总共 52 个，再加上 10 个数字和+和/总计 64 个。

Base64 的大概算法就是将原数据中每个字节（8 位）中的 6 位二进制截取出来转成十进制然后在 Base 64 索引表中找到对应的字符，例如010011转成十进制是19，对应的字符是T，以此类推把剩下的数据进行转换。但是如果是这样的话肯定会出现不够 6 位的情况，假设现在原数据转换完以后多出来 4 位0001，此时需要在原数据中新增一个字节，并从这 8 位中取两位填充 0 然后给这 4 位凑成 6 位，这样就变成000100对应的十进制是4，8 位被取出两位后剩下的 6 位此时刚好转换完成，但是这 6 位没有数据所以整体转换后只用=表示。

用途：

  • 为⼆进制数据扩充了储存和传输途径（例如可以把数据保存到⽂本⽂件、可以通过聊天对话框或短信形式发送⼆进制数据、可以在 URL 中加⼊简单的⼆进制数据）。
  • 普通的字符串在经过 Base64 编码后的结果会变得⾁眼不可读，因此可以适⽤于⼀定条件下的防偷窥（较少⽤）。

Base64 加密文件传输会更安全和⾼效吗？

不安全，Base64 本身不是什么加密技术，而且码表还是公开的；也不高效，Base64 从 8 位转成 6 位会导致数据增⼤ 1/3 ，降低⽹络性能，增⼤⽤户流量开销，是画蛇添⾜的⼿段。如此看来，Base64 除了在一些地方用来将图片变成字符串形式进行传输以外基本没什么其它用途了。

Base58

Base64 还有一个变种是 Base58 ，Base58 是在 Base64 的基础上去掉了几个字符：O、0、I、L、+、/。比特币专门用 Base58 来存储地址。这些改动主要⽬的在于⽤户的便捷性，由于去掉了难以区分的字符，使得 Base58 对于「⼈⼯抄写」更加⽅便。另外，去掉了+、/后也让⼤多数的软件可以⽅便双击选取。

百分号编码

在 URL 中，对⼀些保留字符会使⽤百分号%为前缀进⾏单独编码，以避免出现解析错误。例如，要在http://example.com/users后⾯添加查询字符串，查询name为「Love&Peace」的⽤户，如果直接写成http://example.com/user/?name=Love%Peace，&符号就会被解析为分隔符号，因此需要对它进⾏转码，转码后的 URL 为http://example.com/user/?name=Love%26Peace。这种编码仅⽤于 URL ，⽬的是避免解析错误的出现。

压缩与解压缩

压缩指的是将数据使⽤更具有存储优势的编码算法进⾏编码，简单来说就是换一种形式进行存储，使得空间占用变小。解压缩指的是将压缩数据解码还原成原来的形式。常见的压缩算法：DEFLATE 、JPEG 、MP3 等等，其中 DEFLATE 是 ZIP 的压缩算法。

压缩属于编码吗？

是。所谓编码，即把数据从⼀种形式转换为另⼀种形式。压缩过程属于编码过程，解压缩过程属于解码过程。

媒体数据编解码

图片、音视频的编解码指的是将这些原数据编写成可以存储的格式，例如图像的编码就是将图像数据写成 JPG 、PNG 等文件等编码格式，而图片解码就是将这些格式的文件中的数据解析成标准的图像数据。此外就是有损压缩这个概念，有损压缩是不符合通常意义所说的编码的，因为有损压缩不可逆。

序列化

序列化是指把数据对象（⼀般是内存中的，例如 JVM 中的对象）转换成字节序列的过程。对象在程序内存⾥的存放形式是散乱的（存放在不同的内存区域、并且由引⽤进⾏连接），通过序列化可以把内存中的对象转换成⼀个连续的字节序列，从⽽使⽤ byte[] 等形式进⾏本地存储或⽹络传输。而在需要的时候可以通过反序列化将这些数据读取到内存中来使⽤。

简单来说，内存中的数据存储形式是与操作系统或虚拟机相关的，不是二进制形式，因此不能进行传输和文件存储。而序列化过程把内存对象转变成了二进制形式，就可以进行传输和存储了。

序列化严格意义上来说也不是编码，编码是把数据由⼀种数据格式转换成另⼀种数据格式；⽽序列化是把数据由内存中的对象（⽽不是某种具体的格式）转换成字节序列。

哈希

Hash 可以把任意数据转换成指定⼤⼩范围（通常很⼩，例如 256 字节以内）的数据，相当于从数据中提出摘要信息，因此 Hash 最主要⽤途是数字指纹。

Hash 算法需要注意两点，一是确保低碰撞率，二是防破解，经典的算法有：

  • MD5
  • SHA1
  • SHA256

Hash 的实际用途：

  • 数据完整性验证。例如在下载文件时，上传者可能会提供文件对应的 MD5 或者 SHA1 方便文件下载完成后进行校对。
  • 唯一性验证。例如使用 Java 中的hashCode()。每次重写equals()时都要重写hashCode()，重写hashCode()时把equals()中的每个⽤于判断相等的变量都放进hashCode()中，⼀起⽣成⼀个尽量不会碰撞的整数即可。
  • 快速查找。例如 HashMap 。
  • 隐私保护。当重要数据暴露时，可以选择只暴露其哈希值，例如网站在注册时，键入用户名和密码后，系统在写入数据库时，不会直接明文存储密码，而是会对密码进行哈希，这样就可以保证隐私。同时现在的网站基本上已经没有「找回密码」的功能了，因为数据库存储的是哈希值，哈希值是不可逆的，网站自己都不知道密码是什么。相对的，现在的网站会提供「忘记密码」或者「重制密码」的选项，直接让用户定一个新密码。同时密码如果设置得过于简单或者大众，有可能会被使用彩虹表猜出。而为了对付彩虹表，网站系统往往也会对用户输入的密码加「盐」，例如原密码为「123456」，网站加的盐或许是可变的或许是不可变的，加了盐之后的密码假设是「123456abc」，此时再对密码进行哈希，这样以来想要通过彩虹表破解密码几乎不可能。

Hash 不是编码。因为 Hash 是单向过程，往往是不可逆的，⽆法进⾏逆向恢复操作，此外 Hash 也不是加密，仍然是因为 Hash 是单向过程，⽆法进⾏逆向恢复操作。

字符集

⼀个由整数向现实世界中的⽂字符号的 Map 。

  • ASCII：128 个字符，1 字节
  • ISO-8859-1：对 ASCII 进⾏扩充，1 字节
  • Unicode：13 万个字符，多字节
    • UTF-8：Unicode 的编码分⽀
    • UTF-16 ：Unicode 的编码分⽀
  • GBK / GB2312 / GB18030：中国⾃研标准，多字节，字符集 + 编码
